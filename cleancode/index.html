Clean Code Review

Meaningful Names

1) Variables should be meaningful.

2) Variables should be understandable on its own.

3) Pronounceable names.

4) Avoiding acronyms and confusing names.

Functions

1) Compact and does only one thing.

2) Few number of arguments is better.

3) Desciptive; should be easy to read and understand.

4) Don't use flag arguments.

5) Avoid duplication of methods.

Comments

1) A good code should speak up for itself and doesnt needs comments.

2) Can be used for clarification and explanation of intent of a code.

3) Don't comment out codes. Just remove them.

4) Don't put redundant, useless comments.

Source code structure

1) Related code should be kept vertical.

2) Varaiable declaration should be near their usage.

3) Dependent functions should be near their usage.

4) Functions should be arranged downward depending on usage.

5) Use Spaces between operators, parameters and commas.

6) Keep lines short.

7) Don't use break indention.

Objects and data structures

1) Objects hide the data abstraction and expose methods that operate the data.

2) Data structure expose their data and have no meaningful functions.

3) Avoid making hybrid classes, functions that do significant things .

4) Use procedural or OO method according to the problem in hand.

Error Handling

1) Use Try and catch not and avoid if statements.

2) Make a separation between business logic and error handling code.

3) Create contextual messages about the error

4) Dont Return Null in methods instead of empty object.

5) Dont Pass Null to methods as this generates NullPointerExcetion

Test

1) Make one assert per test.

2) Readable.

3) Independent.

4) Repeatable
